# 2026-02-18

## EP Report Writer Project (Paul)

Paul is building a local AI-powered report writer for Exercise Physiology NDIS support letters.

### Setup
- Proxmox home server with 2x RTX 3060, 64GB RAM
- Ollama running on LXC at 192.168.1.174
- Model: qwen3-vl:30b
- App runs on OpenClaw's Debian host, accessed from Windows PC via browser
- Python3 + Flask, port 5000

### Project files
- `/root/.openclaw/workspace/report-writer/` — full project
- `app.py` — Flask web app with section-by-section generation
- `database.py` — SQLite tracking for generations, edits, feedback
- `clinical_rules.json` — multi-condition clinical rules (SCI, stroke, MS, CP, TBI, amputation, neuromuscular)
- `section_prompts.json` — detailed section prompts with NDIS Section 34 criteria
- `REVIEW.md` — security/code review with prioritised findings

### Key decisions
- Section-by-section generation (not full report at once) works better with smaller models
- Clinical rules baked into prompts prevent hallucination (e.g. AD only for T6+)
- SQLite database tracks all generations, edits, and feedback for continuous improvement
- Feedback loop: edits and feedback are fed back into future prompts
- Removed all branding (Making Strides, NDIS) — generic EP report tool
- Not SCI-specific — supports multiple diagnoses
